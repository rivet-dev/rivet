## Codebase Patterns
- Frontend theme CSS is defined in TWO files that must stay in sync: `frontend/packages/components/public/theme.css` (canonical) and `frontend/src/components/theme.css` (mirror)
- Engine mode frontend routes are limited to `ns.$namespace/index.tsx` and `ns.$namespace/connect.tsx` (cloud mode has more routes)
- Engine admin token for dev is "dev" (set in config.jsonc)
- The engine requires a config file and `start` subcommand: `rivet-engine --config <config.jsonc> start`
- Docker infra for dev-multinode does NOT expose ClickHouse/NATS ports to host; use container IPs or dev-host docker-compose instead
- Namespace creation API uses snake_case fields: `{"display_name":"...","name":"..."}`
- Frontend typecheck command is `pnpm check-types` (not `pnpm typecheck`)
- Both theme.css files now have a Google Fonts @import for IBM Plex Sans/Mono. Font family definitions live in `tailwind-base.ts`.
- Ladle (`.ladle/ladle.css`) has its own `:root` block that must be updated separately when CSS variables change. It does NOT import from the canonical theme.css `:root` block since it forces dark theme by overriding `:root` directly.
- React Flow minimap `maskColor` in `workflow-visualizer.tsx` and `xyflow-nodes.stories.tsx` is hardcoded and must be manually updated when background color changes.
