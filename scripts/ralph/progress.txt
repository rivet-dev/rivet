## Codebase Patterns
- Workflow compaction key helpers live in `engine/packages/depot/src/conveyer/keys.rs`; branch-local manifest/staging keys stay under `BR/{branch_id}/CMP`, while global pin/proof/dirty indexes use reserved partitions `0x70..=0x75`.
- PITR and workflow compaction limit constants live in `engine/packages/depot/src/conveyer/constants.rs` and are re-exported at the `depot` crate root.
- Workflow compaction persisted payload structs live in `engine/packages/depot/src/conveyer/types.rs`; expose vbare encode/decode helpers there and cover them from `engine/packages/depot/tests/`.
- Depot page reads prefer PIDX/DELTA, then latest branch SHARD at or below the read cap, then workflow `CMP/cold_shard` refs and legacy cold manifest layers outside the UDB transaction.
- Workflow compaction signals and durable workflow state live in `engine/packages/depot/src/workflows/compaction.rs`; expose vbare encode/decode helpers there and cover them from `engine/packages/depot/tests/`.
- Workflow compaction branch workflows use `DATABASE_BRANCH_ID_TAG` as their only stable unique tag; the manager stores companion workflow ids before entering its durable loop.
- Workflow compaction dirty admission lives in `engine/packages/depot/src/conveyer/commit.rs`; runtime callers inject a signaler into `Db` to create or target the unique manager workflow.
