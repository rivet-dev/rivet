## How it works

When you create an agent session, agentOS automatically injects a system prompt that tells the agent about its environment. The prompt has two parts:

1. **Base prompt** — describes the VM environment, available runtimes, constraints, CLI tool discovery, and links to documentation
2. **Tool reference** — auto-generated from all registered toolkits, listing every tool with its CLI signature and description

The base prompt lives at `/etc/agentos/instructions.md` inside the VM (read-only). The agent can also read it directly.

Each agent type receives the prompt through its native mechanism:

| Agent | Injection method |
|---|---|
| PI | `--append-system-prompt` CLI flag |
| Claude Code | `--append-system-prompt` CLI flag |
| OpenCode | `OPENCODE_CONTEXTPATHS` env var |

The prompt is always appended to the agent's existing instructions. It never replaces CLAUDE.md, AGENTS.md, or other agent-specific configuration files.

## Tool reference generation

When toolkits are registered (at VM or session level), agentOS introspects their Zod schemas and generates a markdown reference. This includes:

- CLI command signatures with flag names and types
- Required vs optional parameters
- Example invocations from tool definitions

The tool reference is always injected, even when the base OS prompt is disabled.

## Disabling the system prompt

Pass `skipOsInstructions: true` when creating a session to suppress the base OS prompt:

```ts @nocheck
const session = await vm.createSession("pi", {
  env: { ANTHROPIC_API_KEY: process.env.ANTHROPIC_API_KEY! },
  skipOsInstructions: true,
});
```

This skips the base environment description but still injects tool documentation. Tools are unusable without their docs, so the tool reference is always included.

## Custom instructions

Use `additionalInstructions` to append your own context to the system prompt:

```ts @nocheck
const session = await vm.createSession("pi", {
  env: { ANTHROPIC_API_KEY: process.env.ANTHROPIC_API_KEY! },
  additionalInstructions: "Always write tests before implementation.",
});
```

This is appended after the base OS prompt and before the tool reference. User-provided instructions take priority over OS defaults.
