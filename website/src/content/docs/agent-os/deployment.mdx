## Rivet Cloud

The fastest way to deploy. Rivet Cloud handles infrastructure, scaling, and updates.

<CodeGroup>
```ts @nocheck client.ts
import { createClient } from "rivetkit/client";
import type { registry } from "./server";

// Connect to your Rivet Cloud endpoint
const client = createClient<typeof registry>("https://your-project.rivet.dev");
const agent = client.vm.getOrCreate(["my-agent"]);

const session = await agent.createSession("pi", {
  env: { ANTHROPIC_API_KEY: process.env.ANTHROPIC_API_KEY! },
});
await agent.sendPrompt(session.sessionId, "Hello from the cloud!");
```

```ts @nocheck server.ts
import { agentOs } from "rivetkit/agent-os";
import { setup } from "rivetkit";
import common from "@rivet-dev/agent-os-common";
import pi from "@rivet-dev/agent-os-pi";

const vm = agentOs({
  options: { software: [common, pi] },
});

export const registry = setup({ use: { vm } });
registry.start();
```
</CodeGroup>

## Self-hosted

Deploy on your own infrastructure. agentOS runs anywhere that supports the Rivet runtime.

### Requirements

- Node.js 20+ runtime
- SQLite support (bundled with RivetKit)
- Sufficient CPU and memory for agent workloads

### Local development

```bash
npx rivetkit dev
```

### Production deployment

```bash
npx rivetkit deploy
```

## Scaling considerations

- Each agentOS actor runs one VM. Scale by creating more actor instances with different keys.
- Actors sleep after 15 minutes of inactivity (configurable). Sleeping actors consume no compute.
- SQLite data is persisted per actor. Each actor has its own filesystem and session history.
- Use [Queues](/docs/agent-os/queues) to distribute work across multiple agent actors.

## Resource limits

Configure CPU, memory, and other limits per actor. See [Security](/docs/agent-os/security) for details on resource controls.

See [Configuration](/docs/agent-os/configuration) for all available deployment options.
